# Prompt工程

Prompt工程目的 : 設計任務規格
 
在還沒開始學之前需要了解的事

1. 最重要的放前面
2. 重要的事重複說 ( 與1相關都跟權重有關 )
3. 結構化 : 列點
4. 溝通與專案管理能夠幫助拆解工作流程
5. 了解一般的模組是通用模型，並非專業模組，可能會生成錯誤資訊
6. 一個聊天室窗中盡量專注在一個任務，不要混雜更多其他問題，避免混淆視聽。
7. 1比5原則(老師經驗) : 比如要生成1000字就要提供5000字資訊
8. 根據**任務**與**平台**選擇適合的提示詞結構，只要具備邏輯與測試結果符合需求即可。
9. 練習與AI深入溝通，收斂結果，不要太相信第一版回答的內容

***推薦一些可用工具***

- 查證AI工具 : Perplexity、Felo => 使用爬蟲
- 圖表生成工具 : napkin
- 語音轉文字 : SRT檔 ( 時間線、標示發言者 )
- 設計格式 : YAML

# 一次性明確提示語-VISTA

## VISION: 你想要達成什麼結果? **可檢視**的結果

- 幫我準備面試[X]
- 我想要準備一場軟體工程師的人員面試，希望整理出常見的面試題回答範例語表現建議[O]
但以上兩點都非常模糊...

## INFORMATION-需要更多背景資料

- 你的身分
- 對象是誰
- 已知條件
- 可用資源
- 重要限制
- 目前狀態
- 參考資料
- 人、事、時、地、物、材
- EX. 我是剛畢業的學生，沒有正式的工作經驗，有一些產學合作遊戲前端開發的經驗，個性比較內向，要應徵軟體工程師。

## STRATEGY-已經試過什麼偏好什麼?

- 已經做過哪些嘗試?
- 哪些方法有效無效?
- **必須避開那些?**
- 建議 : 不要把自己偏好帶入 ( ex.你喜歡什麼... )
- EX. 我之前查過面試題，但回答很像背稿，我希望有自己的故事背景且真誠，不要太誇張。

## TIMELINE-時間和節奏

- 期限、可投入的時間
- 階段安排
- 優先順序
- 是否需要短期版本
- EX. 我三天後要面試，每天可以準備30分鐘，希望聚焦在最終要的題目

## ASK-明確告訴AI怎麼輸出

- 表格、條列
- MARKDOWN、HTML
- 清單、腳本
- 簡報大綱、檢查列表、WORD格式?
- EX.我希望列出五個常見得面試問題，提供自然版的回答技巧與範例，用條列式呈現。

## 將以上條列式VISTA列點

使用上面的條例文字後，提示詞可以這樣打 : 

```
- Vision : 我想要準備一場軟體工程師的人員面試，希望整理出常見的面試題回答範例語表現建議。
- Information : 我是剛畢業的學生，沒有正式的工作經驗，有一些產學合作遊戲前端開發的經驗，個性比較內向，要應徵軟體工程師。
- Strategy : 我之前查過面試題，但回答很像背稿，我希望有自己的故事背景且真誠，不要太誇張。 
- Timeline : 我三天後要面試，每天可以準備30分鐘，希望聚焦在最終要的題目。 
- Ask : 我希望列出五個常見得面試問題，提供自然版的回答技巧與範例，用條列式呈現。
```

任務可能包含許多流程，比如需要 : 

1. 目標、定位 : 相關數據 ( 像是利用perplexity查詢資訊 )

# 製作一個專業助理-複雜案例建議結構 

可帶入思考的流程與prompt架構 : 

**角色設定 &rArr; 背景資訊 &rArr; 任務拆解 &rArr; 操作步驟 &rArr; 限制條件 &rArr; 輸出格式 &rArr; 驗收標準(什麼時候結束) &rArr; 例外處理**

## 1. 角色設定

三種對象要分清楚 : AI、使用者、受眾

- AI : 他要扮演誰? (以職位或是專業說明or實際作為成果之類的)
- 使用者 : 誰會操作AI?
- 受眾 : 產出給誰看? 誰驗收? (利害關係者)

但...如何了解AI有哪些角色可以扮演呢?
我們可以嘗試從資料庫中抓取相關有用的角色資訊。我們向下面這樣寫 : 
```我想要撰寫與請教行銷社群的相關文章，請問AI可以扮演那些角色幫忙，這個角色有哪些優勢，請列出三個。```

- 注意1 : 不要太快接受AI第一次給你的資訊，可以重複驗證或深入了解
- 注意2 : 要跟AI明確說明文字是要用到哪裡去的? (EX.你想要打包目前的資料結果，要明確告知是要在新視窗中使用的資訊)

## 2. 提供AI判斷所需的背景

- 使用情境
- 任務來源
- 對象條件
- 已有資料

比如說履歷的話 : 基本資料、**軟實力**、**硬實力**、確認職位

## 3. 將模糊的事情定義清楚

當你有資料時，不一定要直接用PDF或Word，更好的做法是 : 

- 整理標題或短落
- 轉換成列點式
- 必要的表格與資料來源 
- 直接給**使用資料範例**

## 5. 限制條件

AI要做得好就是要把限制做好，常見的限制用詞如下 : 

- 若資料不足，請標註"待補"
- 僅能重新整理
- 不能杜撰數據內容
- 不得捏造人名/來源
- 不要使用專業術語
- **請使用台灣用語**
- 個資請提醒使用者刪除

## 6. 完成的標準

沒有驗收標準就很難判斷好不好，基本上就是讓AI做迭代

## 用來更明確定義目標的做法 - Few Shot 示範 ( Few-Shot Demonstration )

**利用少量的資訊讓AI清楚了解接下來的目標。**可以做以下的練習 : 
```
你是［角色］，我將依序提供多組不同風格的「輸入文字」與「對應的輸出範例」。
你的任務是：
1. 觀察並歸納每組範例的文字風格、結構、語氣與特色。
2.每看完一組範例，先用你自己的話總結它的風格特徵，並問我是否正確，以及哪些部分是我喜歡的、哪些我不喜歡。
3.當我提供所有範例並完成確認後，你需整合我喜歡的特徵，形成一份「最終風格指南」。
4.等我給你「原始文字」時，你要依據這份最終風格指南來改寫或生成內容。
如果理解，請先回覆「我已準備好接收第一組範例」。
```

# Refine prompts 提示詞模組化

## 提示詞模組化目的

- 可重複使用
- 可逐段修改
- 可交付
- 可測試與迭代
- 可使用gems/GTPs

## 相關範例

**協助幫忙修正提示詞小幫手**
但其實比較好做法是直接寫在AI個性中。但若沒有使用AGENT的情況下可以使用以下範例:

```
你現在是一位「跨平台 Prompt 優化顧問」，精通不同 AI 平台（例如 ChatGPT、Claude、Midjourney、DALL·E、Gemini、Notion AI 等）的語法特性與最佳實踐。
目標：協助使用者針對不同平台需求，將原始 prompt 逐步修正並優化（Refine Prompts 模式），直到符合該平台最佳效果。
=== 使用流程 ===
1. **確認平台與用途**
   - 詢問：「請問你要在哪個平台使用這個 prompt？該平台的用途是什麼？」
   - 確認該平台的特殊要求（格式、語法、輸出特性、限制等）。
2. **取得初版 Prompt**
   - 請使用者提供原始需求或現有 prompt。
   - 若資訊不足，詢問背景、情境、限制條件、期望輸出範例。
3. **平台化轉換（Platform Adaptation）**
   - 根據平台特性，將初版 prompt 調整成該平台可執行的版本。
   - 說明每個修改的原因與作用。
4. **提示優化模式（Refine Prompts Loop）**
   - 在每次輸出後，詢問：
     「是否需要針對清晰度、精準度或風格進一步優化？」
   - 提供 2–3 個可選的優化方向（例如：加限制條件、加入範例輸出、改變語氣）。
   - 根據使用者選擇進行更新，直到滿意為止。
5. **最終確認與測試**
   - 提供最終版本 prompt。
   - 附上簡短說明：
     - 平台適配度
     - 可達成的效果
     - 潛在限制
=== 規則 ===
- 每一步都要互動式詢問使用者再繼續。
- 優化時要保留原本的核心需求，不偏離目標。
- 可以主動提醒使用者該平台的最佳實踐或注意事項。
- 若使用者需求過於模糊，先引導補充再進行轉換。
開始前請先問我：
「你要在哪個平台使用這個 prompt？該平台的用途是什麼？」
```

**確認任務的小助手**

```
你現在是一個專業顧問，我有一個任務：【在這裡輸入你的目標任務，例如：幫我選出最適合的行銷方案】。
請使用「內心獨白（Internal Monologue）」的方式，逐步展示你的思考過程與理由，並在每個階段清楚標示：
思考過程（內心獨白）：
你目前在考慮什麼？
為什麼這樣想？
有哪些假設或可能的限制？
中間結論（本階段的暫時判斷）
問我：「要不要針對某個面向深入分析？」
重複以上步驟直到所有面向分析完成，最後：
 4. 最終結論 / 建議：
彙整全部推理與分析結果
提供可行方案與比較理由
如果有風險或不確定性，請說明如何降低或監測
```

**拆解大任務的提問(Step by step)**

```
[目標] 你是[角色]，請使用「思維鍊方式」（將大問題拆成多個小問題，一步步詢問）逐步向我提問。

[操作步驟]  
1. 每次只問一個問題。  
2. 在我回答後，先確認答案無誤再進下一題。  
3. 若需要，可以針對我的回答直接提出修改建議。  

[注意]  
- 持續循環上述步驟，直到你獲得足夠的資訊以完成最終結果或達成限制條件。  

[範例]  
你：你想解決的主要問題是什麼？  
我：增加網站流量。  
你：你的網站類型是什麼？（進入下一個問題）
```

