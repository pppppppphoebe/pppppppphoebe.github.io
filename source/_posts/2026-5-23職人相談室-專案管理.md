### 什麼是專案?
為特定目標而進行的臨時性工作。
- 有明確目標
- 有開始與結束

### 為什麼要學專案管理?
- 有效率作專案的方法論。

### 專案鐵三角思維 
ref:<https://asana.com/zh-tw/resources/project-management-triangle>
{% asset_img "鐵三角.png" "專案鐵三角" %}
核心概念一句話：

**好、快、便宜，三個你只能選兩個。**

- 你要 快 又 好 → 成本一定高（加人、加資源）
- 你要 便宜 又 好 → 時間要拉長
- 你要 快 又 便宜 → 品質會犧牲（scope砍掉很多）

跟專案基線的關係

**前面說的「三條基線」——範疇、時程、成本——就是鐵三角的三個角，而 品質在正中間，受到三個角的擠壓。**

實際例子：

你設定了專案基線（存檔點）：

- 範疇：蓋一棟兩層樓房子
- 時程：6 個月
- 成本：1000 萬

到了第 3 個月，業主說：「我想加一個地下室。」

這時候鐵三角就來了：

- 範疇變大了（多了地下室）
- 要不就 時間延長（8 個月）
- 要不就 成本增加（1500 萬）
- 如果時間跟成本都不變，品質就會被犧牲（地下室亂蓋）

所以專案經理會說：

「要加地下室可以，但我們需要走變更流程，調整基線——要嘛加錢，要嘛延時間，你選。」

你該記住的三句話

1. 鐵三角告訴你：專案沒有免費的午餐
2. 基線告訴你：你到底承諾了什麼
3. 兩個一起用：當有人想改東西，你就知道要怎麼談判

### 專案基線

**專案基線 = 你的專案「原始版本」**
想像你在玩一個 RPG 遊戲，在進入迷宮之前先存一個檔。萬一走錯路、被怪打死、選錯對話選項，你都可以「讀取進度」回到那個時間點。

專案基線就是那個「存檔點」。

具體來說，有三條主要的基線：

1. 範疇基線（Scope Baseline） — 一開始說好「要做哪些事」
    - 專案章程、**WBS（工作分解結構）**這些
2. 時程基線（Schedule Baseline） — 一開始說好「什麼時候做完」
    - 就是甘特圖上那些時間點、里程碑
3. 成本基線（Cost Baseline） — 一開始說好「要花多少錢」
    - 預算分配

合起來叫 績效衡量基線（Performance Measurement Baseline, PMB）。

**為什麼重要？**

當專案進行中有人說「我們加個功能吧！」或者「進度延遲了」，你拿出基線來對比，就知道：
- 跟原本講好的差了多少
- 需要做「變更管理」— 不是直接改，而是走正式的變更流程
- 專案經理才能回答老闆：「我們原本說好三個月、一百萬、做這十件事，現在要加兩件事，需要加兩週跟二十萬。」

### 成為PM的關鍵

- 識別個人若項
- 制定練習計畫
- 找到對的賽道
- 持續反覆改進

### 常見任務

- 需求整理 : 將模糊的需求整理成具體可執行的任務描述
- 時程規劃 : 制定專案里程碑、分配資源、交付時程、持續追蹤
- 風險管理/應變 : 對應計畫
但每個產業的需求差異很大

### 核心能力-數位力 

#### 1. 進度追蹤與任務管理

核心需求：誰、什麼時候、做完什麼、下一階段的阻礙是什麼

| 工具 | 適合情境| 特點|
| ---------------------------------- | ------- | ------------------------------------ |
| Notion| 個人／小團隊  | 極靈活，可自訂資料庫視圖（表格、看板、行事曆、時間軸）|
| Asana| 中大型團隊   | 時間軸 + 任務依賴關係強，適合有明確里程碑的專案|
| Trello| 輕量／看板流  | 簡單直覺，適合 Kanban 方法論|
| Jira| 軟體開發    | 敏捷開發標配，Sprint、Backlog、Burndown Chart |
| ClickUp| 什麼都要的團隊 | 功能最雜但最全，有目標 OKR + 文檔 + 白板 + 時間追蹤|
| Monday.com| 中大型企業   | 視覺化極強，老闆簡報友善|
| Microsoft Project / Project Online | 傳統專案| 正式排程、關鍵路徑、資源撫平，老派但權威|
| Smartsheet| Excel 控 | 看起來像 Excel 但其實是專案管理工具|

實務建議

- 個人／小團隊 &rArr Notion 就夠了
- 多人協作有 deadline 壓力 &rArr Asana 或 ClickUp
- 公司強制用什麼就用什麼，不用煩惱 

#### 2. 資訊整理與數據分析

核心需求：會議記錄、決策脈絡、專案數據可視化

| 工具| 適合情境| 特點|
| ------------------------------------------------------- | ------------- | ------------------------------ |
| Notion| 幾乎所有知識管理      | 文件 + 資料庫 + Wiki 一體，可以關聯任務和會議記錄 |
| Confluence| Atlassian 生態系 | 跟 Jira 深度整合，文件版控強|
| Google Workspace (Docs / Sheets / Slides)| 協作首選| 同步編輯、評論、版本歷史，幾乎無痛|
| Microsoft 365 (Word / Excel / PPT + SharePoint / Teams) | 企業標準| 權限管理強，跟 Outlook / Teams 整合|
| Miro / Mural| 腦力激盪、流程圖| 白板協作，拆需求、畫流程圖超好用|
| Tableau / Power BI| 大專案數據分析| 專案追蹤數據做成儀表板，給老闆看的好東西|
| Excel + 樞紐分析表| 萬用最後手段| 便宜、萬能、人人會，但容易失控|

實務建議

- 會議記錄 &rArr Notion 或 Google Docs
- 流程圖 &rArr Miro
- 數據給老闆看 &rArr Power BI 或 Excel 圖表

#### 3. AI輔助工具

核心需求：加快重複性工作、產出草稿、摘要會議、預測風險

| 工具                      | 用途                                                |
| -----------------------   | ------------------------------------------------- |
| ChatGPT / Claude          | 產出 WBS、會議記錄模板、風險清單、Email 草稿、專案計畫初稿 |
| Notion AI                 | 直接在 Notion 裡摘要、翻譯、改寫、生成任務 |
| MS Copilot                | 在 M365 生態裡整理 Teams 對話摘要、生成 Excel 公式、PowerPoint 排版 |
| Otter.ai / Fireflies.ai   | 自動錄音 + 逐字稿 + AI 摘要會議重點 |
| Gamma / Beautiful.ai      | AI 自動生成簡報，輸入大綱就幫你排版 |
| Taskade                   | AI 代理可自動分派任務與進度提醒 |
| Linear                    | 工程團隊用，AI 幫你排優先級和估算工時 |
