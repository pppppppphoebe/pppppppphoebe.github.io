職涯課程 - 履歷與面試

Author: HsingYin Hsieh Date: April 26, 2026  9:28:13 Category: Work Log Daily

履歷課內容

投遞履歷與面試思維

  • 意願、態度
  • 沒有亮點 -> 轉譯!!!
  • 緊張 -> 行動、準備
  • 不讀取就不要再追ㄌ QQ
  • 核心! : 帶給公司信任感

HR履歷亮點

  • 職稱、技能
  • 成果數據
  • 版面整齊、清楚

產品說明書思維

履歷不是回憶錄，是解決企業問題的產品 :

  • 自傳直接切入最新的工作經驗 or 學歷和技能
  • 我能為你做什麼
  • 針對職缺需求呈現:
    • 先做公版(不需要挑職位的履歷)
    • 再針對專職細作
  • 履歷左上角是最重要的區域
  • 會技能雖然不能達到業界水準，但相較於其他人就有機會(點單來說也要懂包裝)

練習:我的核心技能??

關於我自己 :

  1. 2D遊戲前端開發，建置UI、動畫調整、與封包串接
  2. 簡單的SQL語言: CRUD
  3. Python AI模型建置訓練以及使用OpenCV分析影像數據
  4. 靜態網頁前端設計包含網頁架設

更詳細建議 :

  • 會用這個技能做到什麼??
  • 時間長度? 持續有在使用更能夠讓人相信
  • 強度要講出來 -> 案例:用這個技能做什麼?
  • 整理專案經歷的小故事

數據化經歷

  • 這個沒有數據(吐血)
  • 這個數據的脈絡:
    • 怎麼執行的?
    • 怎麼處理過程中的問題?

跨領域轉職&能力轉譯

  • 可遷移的核心能力 : 專案管理、資料分析、溝通、數位工具

  • 展現你的決心 : 曾經做了什麼?

  • 為什麼?動機?優勢是甚麼?

  • 重點是用這個工具/能力做了什麼? >> 案例

  • 我的軟實力 : 負責任 ?

    • 事件 : 產學合作項目能在時程內按時完成工作細項、並且定期與公司確認進度與回報相關問題，像是美術素材缺失或是需要補足一些封標資料。

  • 可能會被問到的問題?

    • 試著去了解答案
    • 然後再問主管或相關有經驗的同事
    • X : 直接問主管可能會被認為要把資料丟出去，太極端都不好。

工作經歷不多?

  • 從課程、競賽、實作(作品集)
  • 過程艱辛 -> 但克服了
  • 結果很好

履歷篩選原則

  • 與職缺的相關經驗
  • 低關聯性只會稀釋重點不要寫

自傳模組

  • 兩個重點 :
    • 做什麼事獲得很大的成就感
    • 做什麼遇到什麼困難和如何解決
  • 階段一: 我是誰?
  • 階段二: 我的代表成就?
  • 階段三: 為甚麼選這家公司?
  • swot分析
    • s: 技能經歷個人特質優勢
    • w: 劣勢->再補強哪個缺口
    • o: 在哪個產有有力?
    • t: 競爭者有什麼優勢?
  • 興趣絕對不可以當作原因…

關鍵字精準打擊

  • ATS應徵者追蹤系統也有面向企業
  • 針對JD來做修改
  • 多版本履歷管理

履歷地雷

  • 錯字、語病
  • 不專業照片 - 台灣一定要附照片
  • 一頁就好
  • 千萬不要負面言論…

求職信

  • 開頭 : 為什麼要選這家公司(具體)
  • 中段 : 與職業的關聯、與能夠帶來什麼價值?
  • 結尾 : 積極表達面試意願!

LINKEDIN經營

  • 專業大頭貼
  • 加入核心價值
  • 主動連結
  • 分享工作心得!

初步檢驗

  • 前1/3是否有清楚的職稱與技能標籤
  • 每段經歷是否包含至少一個數據獲具體成果
  • 版面是否整潔，有適當留白?
  • 是否有針對這次職缺調整關鍵字?
  • 自傳是否使用列點式而非流水帳?
  • 移除低關聯性的經歷?

一些其他問題

  • 態度展現 => 我現在已經做到什麼了展現持續進步的可能

面試課內容

生理緊張可以做的事

  1. 呼吸法 - 吸氣(4sec)憋氣(7sec)吐氣(8sec)，重複三次
  2. 插腰挺胸站立
  3. 自我介紹再練一次XD

專業儀表 - 合宜就好

至少要比公司文化再正式一些。

需要注意的細節

  • 準時 : 不要太早也不要遲到，大約提前10分鐘到剛好。
  • 禮貌 : 從接待人員就要真誠問好。
  • 眼神接觸 : 盡量看對方的鼻子，70%時間維持眼神接觸。
  • 肢體接觸 : 坐直，手自然放著就好。

一分鐘自我介紹

  • 大致架構 :

    • 過去(30sec) : 最近的職涯經歷
    • 現在(20sec) : 目前的能力與職能狀態
    • 未來(10sec) : 為甚麼投遞這個職缺? 目標與熱情

  • 我的版本 :

    • 想應徵的方向 : 應徵Web開發工程師
    • 過去 : 您好，我畢業於台科大資工所，在校期間我曾參與產學合作專案，負責2d遊戲的前端開發，完成兩款遊戲的前端流程與測試。
    • 現在 : 目前我習慣使用Git版本控制專案內容，也在具備Python、C#、Typescript的開發能力，同時也在學習其他前端框架的應用。
    • 未來 : 因為曾經有前端開發以及資料庫的相關經驗，能夠快速學習快速上手工作，期望可以加入公司產出價值。

  • 其他要注意的點 :

    • 其實最重要的是想做什麼，現在和未來要連起來。
    • 說沒有經驗是可以的，但要拉回來!以我的相關經驗佐證。

STAR原則

  • 原則內容

    • Situation : 背景?
    • Task : 需要完成什麼目標?
    • Action : 做了哪些事?
    • Result : 結果/數字?

  • 面試常見的相關問題:
    (可以自己多準備幾個，五個，可以應用在各種情境)

    • 遇到挫折
    • 成長經驗
    • 快速學習與適應力
      “我雖然沒有直接經驗，但我在X中展現快速掌握Y的能力，相信我在3個月內可以達到目標….”
    • 跨領域快速上手

年輕應徵者的四大條件

  1. 穩定度: 人資會期望三年XD
  2. 文化適應
  3. 溝通效率
  4. 自主學習
    面試中，每個回答都暗示符合這四個條件的其中一項。

回答不出來怎麼辦

  • 想一下(5-10sec)

  • 表達邊界 : “目前我的經驗沒有涵蓋這個，但我的作法是…”

  • 展示學習意願 : 如何快速學習填補知識缺口。

  • 其他 :

    • 被勸退怎麼辦?
      真的想要就要展現自己的態度!!!!
      比如 : 職缺在北部但我住在南部怎麼辦? => 老師說: 我已經找好房子了XDDD

為甚麼要錄取你…?

公式組合: 獨特組合 + 具體效果 + 對職位的意義

  • 企業要的不是單獨技能，是要能夠知道螺絲在哪裡並且修好他的人。
  • 基本上就是 : 硬實力 > 跨域附值 > 商業思維

如何說缺點?

  • 也不要說自己沒有缺點
  • 公式 : 真實缺點 + 你已採取的改進行動 + 目前進展
    • 描述真實缺點時要說明目標是怎樣，還沒達成這樣
    • 改進的行動必須要夠多，可以把一些比較日常付出也加入，必須夠有信服力。

離職原因的藝術

避開負面情緒，強調”尋求更大價值的發揮”。

  • 注意不要抱怨前公司或前主管
  • 例如: “希望能尋找發揮XX技能的環境”

壓力測試的應對

  • 思考目的 : 考你的臨場反應與情緒穩定性，不是標準答案。
  • 暫停換氣
  • 結構化回答 : 有邏輯的排序。

五年規劃…

這個問題是你展示你的野心與穩定度是否平衡。

  1. 第一年: 快速上手
  2. 第二年: 深化技能
  3. 第三年: 承擔與領導

關於薪資與福利

  1. 事前調查
    • 查詢同職位、產業、地區的薪資中位數
    • 資料來源: 104(數字可能偏高，因為數據大部分基於大公司資料)、1111(比較廣)
  2. 開口的機會
    • 大概可以問 : 我這樣的薪資職位薪資大概在哪個區間?

沒有下文怎麼辦?

  1. 24小時內感謝信 : 點短3-5句並重申熱情與適合度。
  2. 一周後禮貌追蹤
    • 可以直接問他我的問題在哪邊，畢竟壓力在他身上，然後再重投一次都是可以的!
  3. 覆盤這次面試
  4. 接受結果，繼續前進

刁鑽問題 0..0

  1. 如何證明你不會工作不到一年就離職?
    • 測試穩定度與真實動機
    • 如何回答 : 從職涯規劃與在公司的具體目標，不要空談承諾
    • 其他回應 : “確實年輕人工作穩定性不高(不要反駁他)，但是我不一樣….”
  2. 需要即戰力、為甚麼要花費時間培訓你?
    • 說明自己的學習能力與實際經歷佐證
  3. 薪水不高但是要配合加班?
    • 對報酬的看法與穩定性
    • 可以問的問題(一) : 那我想問一下頻率是怎樣?
  4. 講出三個缺點
  5. 有沒有在同時投遞其他間公司?
    • 可以這樣回答 : 確實有在投遞蠻多家的，因為我現在正在找工作，但今天聽完講解之後對X公司很有興趣:D。

可以問公司的問題

  1. 關於成長 : 公司職位的同事需要花多少時間獨當一面?
  2. 關於職務 : 這個工作最重要的績效是什麼?工作內容是?
  3. 文化 : 在這個公司工作的優秀員工通常有什麼人格特質?

率取後的職涯經營

如何在公司長期發展?

  1. 試用期(3月):理解業務、準時交付、建立信任關係
  2. 熟悉期(4~5月)
  3. 貢獻期(6~12月)
  4. 晉升期(1~2年)
