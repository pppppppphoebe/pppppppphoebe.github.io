LeetCode Review: Heap (Priority Queue) vs. Sort

Author: HsingYin Hsieh Date: June 21, 2026  11:57:57 Category: LeetCode Review

Heap

核心概念

  • 動態維護極值得序列
  • 完全二元樹
  • 局部有序：它只保證樹根（Root）永遠是最大值（Max-Heap）或最小值（Min-Heap），但左右子樹之間沒有特定的順序關係。
  • Priority Queue 是一種抽象資料型態（ADT），而 Heap 是實現它最有效率的方式

使用情境

  • 當你不需要資料完全排好序，而是需要動態、頻繁地獲取極值（最大或最小）。
    • 串流資料（Streaming Data）：資料是一筆一筆進來的，你需要隨時知道目前為止的前 K 大元素（Kth Largest Element）。
    • 圖形演算法：Dijkstra 找最短路徑、Prim 找最小生成樹（用來動態挑選權重最小的邊）。
    • 工作排程（Task Scheduling）：根據任務的優先權（Priority）決定下一個要執行哪一個任務。

Sort

核心概念

  • Sort 是一個將一組混亂的資料依照特定順序（升冪或降冪）重新排列的程序/動作。

使用情境

  • 當你需要整份資料完全有序，或者需要利用順序特性進行後續操作。
    • 二分搜尋（Binary Search）：資料必須先排序才能使用。
    • 計算中位數（Median）或百分位數：需要知道全局的精確位置。
    • 資料報表呈現：例如依據分數從高到低列出所有學生的成績單

複雜度 - 以 Quick/Merge/Heap Sort 為例

  • build: O( NlogN )
  • peek: O(1)
  • insert: O(N)維持有序插入的話
  • extract: O(N)
  • 空間複雜度: O(1)~ O(N)視不同演算法而定

優缺點

優點

  • 全局有序：排序一次後，你可以隨意存取任意排名的元素（例如直接拿 data[i]）。

缺點

  • 靜態：如果資料經常變動（一直有新資料加入），每次都重新排序的成本太高 O(N log N)。

常見問題

Top K Elements / Kth Largest Element (LeetCode 215)

  • 題目：在一堆無序數字中，找出第 K 大的數。
  • 盲點：很多人直覺會想直接 Sort 然後取 data[-K]，時間是 O(N log N)。
  • 最佳解：維持一個大小為 K 的 Min-Heap。遍歷整個陣列，當 Heap 大小超過 K 就 pop。最後 Heap 的頂端就是答案。時間複雜度降到 O(N log K)，在 K << N 時效率遠勝 Sort。

Find Median from Data Stream (LeetCode 295)

  • 題目：資料源源不絕進來，要隨時能取得目前所有數字的中位數。
  • 解法：用 Sort 的話每次都要重新排，會逾時。正確做法是維護兩個 Heap：一個 Max-Heap 存較小的那半部數字，一個 Min-Heap 存較大的那半部數字，調整兩邊平衡，中位數就會在兩個 Heap 的頂端。

Merge K Sorted Lists (LeetCode 23)

  • 題目：將 K 個已經排好序的鏈結串列（Linked List）合併成一個。
  • 解法：將每個串列的頭節點丟進 Min-Heap 中，每次 pop 出最小的接上新串列，並把該節點的下一個節點再 push 進 Heap。
